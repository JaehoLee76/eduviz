# EduViz — 기술문서 & 사용자 매뉴얼

> **EduViz**는 수학·알고리즘·물리·미적분을 **직접 조작하는 인터랙티브 애니메이션**으로 배우는 무료 교육 웹사이트입니다.
> 라이브: **https://jaeholee76.github.io/eduviz/** · 로그인·설치 없이 브라우저로 바로 열립니다.
>
> 이 문서는 ①처음 쓰는 사용자를 위한 **사용 매뉴얼**과 ②개발/운영을 위한 **기술 참고자료**를 겸합니다.
> *최종 갱신: 2026-06-27*

| **알고리즘** | 8장 (뼈대 42 + 심화 268) | **왼쪽 코드에 실행 줄커서 + 오른쪽 캔버스 단계 애니메이션** 쌍 |
| **물리학** | 14장 · 70장면 (+심화 29) | 자체 물리엔진(PhysLab)으로 F=ma를 매 프레임 적분 → 슬라이더·드래그로 실시간 시뮬 |
| **미적분학** | 15장 · 본문 78 + 심화 31 = 109장면 | 수치미분·리만합·오일러법으로 실시간 계산(공식 베끼기 아님) |

### 2.2 공통 특징
- **시네마틱 인트로**: 각 트랙 첫 화면은 인물(뉴턴·아인슈타인·가우스·에이다 러브레이스) 초상화와 함께하는 짧은 애니메이션 → 엔드카드(설명 + 학습 시작/다시 보기).
- **심화학습 분기**: 각 핵심 장면에서 "📚 심화학습" 버튼으로 더 깊은 주제로 분기.
- **학습 패널**: 핵심 요약·연습문제(풀이 토글)·한·영 병기.
- **AI 튜터**: 우상단 "🤖 AI 질문"(단축키 `i`) — 5장 참조.
- **내 기록**: 학습 위치 자동 저장(재방문 시 이어보기), 장면별 메모, AI 대화 기억 — 6장 참조.

## 3. 사용법 (Usage)

### 3.1 시작
1. https://jaeholee76.github.io/eduviz/ 접속 → 홈에서 트랙(수학/알고리즘/물리/미적분) 선택.
2. 첫 인트로 애니메이션은 **클릭하면 건너뛰기**, 끝나면 **▶ 학습 시작**.
3. 좌상단 **← 홈** 으로 언제든 홈 복귀.

### 3.2 화면 구성
- **오른쪽 = 캔버스**: 그래프·도형·시뮬레이션·알고리즘 애니메이션.
- **왼쪽 = 설명**: 제목·본문·핵심 요약·(알고리즘은) 코드 패널. 마스코트 말풍선.
- **아래 = 조작 바**: 이전/다음, 단계 진행, 자동 재생 등.
- **우상단 = 도구**: 🤖 AI 질문 · 📝 메모 · (예정) 로그인.

### 3.3 조작 키 (트랙별)
| 키/버튼 | 기능 |
|---|---|
| **← / →** (또는 이전/다음 버튼) | 장면 이동 |
| **D** | 한 단계 진행 (수학=설명단위, 알고리즘=코드 줄/애니 한 칸) |
| **S** | 자동 재생 |
| **X** | 처음으로(현재 장면 리셋) |
| **R** | 풀이 보기/숨기기 (연습문제) |
| **W** | 학습 패널 열기 |
| **슬라이더** | 값을 바꿔 그래프·시뮬 실시간 변화 (키보드 A/D·F/H 자동 배정) |
| **드래그** | 점·물체를 직접 끌기 (해당 장면) |
| **📚 심화학습(N)** | 더 깊은 분기로 |
| **물리 재생바** | `Space`=재생/정지 · `,`=한 칸 뒤로 · `.`=한 칸 앞으로 · `0`=처음으로 |
| **i** | AI 질문 열기/닫기 |
| **📝** | 이 장면 메모 |

> 화면의 **주황색 알약(힌트)** 이 현재 장면에서 쓸 수 있는 키를 안내합니다.

## 4. 구현 수준 (Implementation)

### 4.1 기술 스택
- **프론트엔드**: 순수 정적 HTML/CSS/**바닐라 JavaScript**. 빌드 도구·프레임워크 없음 → 파일을 브라우저로 열면 바로 동작.
- **공유 엔진**: `js/engine.js`(SceneManager + 단계 기계 + 렌더 루프). 4트랙이 한 엔진을 공유.
- **콘텐츠 분리**: *동작*(`js/content_*.js`)과 *텍스트*(`content/*.json`)를 분리 — 텍스트 수정은 JSON만, 동작은 JS만.
- **물리엔진**: `js/physlab.js` — 반암시적 오일러 적분으로 F=ma를 매 프레임 풀어 운동을 *생성*.
- **호스팅**: GitHub Pages(정적). 배포 = git push → Pages 빌드.
- **백엔드**(AI·계정용): Cloudflare Worker 서버리스 — 8장 참조.

### 4.2 완성도
- 알고리즘 애니메이션 **100%(161/161)** + 심화 14섹션 재구조화 완료.
- 수학 전 26장 전수 감사·보강 완료(빈 화면·정적 장식·골든룰 위반·죽은 코드 0).
- 물리 14장 70장면 + 심화 29분기 완료.
- 미적분 15장 109장면 완료.
- 전 트랙 **수식 표기 정밀 점검 완료**(계수 1/−1/0 처리, 곱셈·함수적용 구분).
- AI 튜터·메모·학습위치 복원 **구현·배포 완료**(라이브 작동).
- 로그인(구글) **코드 완료, Client ID 발급 대기**(발급 시 사용자별 클라우드 동기화 활성).

## 5. AI 질문·답변 — 방식 · 한계 · 품질 (숨김없이)

### 5.1 어떻게 동작하나
1. 우상단 **🤖 AI 질문**(또는 `i`)을 누르고 질문을 입력합니다.
2. 사이트가 **지금 보고 있는 장면의 맥락**(제목·설명·핵심요약·현재 단계·연습문제·코드)과 **최근 대화**를 함께 AI에 보냅니다.
3. AI가 **한국어 존댓말로, 그 장면 주제에 맞춰 간결하게**(보통 3~6문장) 답합니다. 주제를 벗어난 질문은 이 장면으로 정중히 되돌립니다.

### 5.2 어떤 AI인가
- **모델**: Cloudflare Workers AI의 **Llama 3.3 70B**(메타의 오픈 모델).
- **위치**: 답변은 Cloudflare 자체 GPU에서 생성됩니다(외부 API로 나가지 않음).

### 5.3 한계 — 반드시 알아두세요
- **완벽한 정답기가 아닙니다.** Llama 3.3 70B는 **우수한 오픈 모델이지만 최상위 유료 모델(GPT-4·Claude 등)보다는 약합니다.** 깊거나 까다로운 질문, 복잡한 계산·증명에서는 **얕거나 틀린 답**을 할 수 있습니다.
- **환각(그럴듯한 오답) 가능.** "확실하지 않으면 추측하지 말라"고 지시해 두었지만 **완전히 막지는 못합니다.** 중요한 사실·수치는 **반드시 화면의 실제 콘텐츠로 교차 확인**하세요.
- **대화 기억은 제한적**입니다: 같은 장면에서 **최근 대화는 자세히, 그 이전은 요약**해서 기억하며(무한 아님), **장면이 바뀌면 새 대화**로 시작합니다.
- **질문 횟수 제한(무료 한도)**: 현재는 **사이트 전체가 무료 한도를 공유**합니다. 많이 쓰이면 "토큰충전 남은시간"이 뜨고, 매일 리셋(협정세계시 0시)됩니다. (로그인 붙이면 개인별 할당으로 확장 예정.)
- **로그인 전**: 학습 기록·메모·대화 기억이 **이 브라우저에만** 저장됩니다(다른 기기엔 안 따라옴).

### 5.4 잘하는 것 / 권장 사용
- ✅ **지금 장면을 더 쉽게 풀어 설명**, 헷갈리는 부분 다시 묻기, 직관·비유, 가벼운 확장("실생활 예는?").
- ⚠️ **정밀한 수치·증명·시험 정답**은 보조 참고로만 쓰고, 핵심은 화면의 실측 콘텐츠와 교과서로 확정하세요.

## 6. 내 기록 (학습 위치 · 메모 · 대화)
- **학습 위치 자동 저장**: 보던 장면을 기억 → 다시 오면 "이어서 학습 →" 로 바로 점프.
- **장면별 메모**: 📝 버튼으로 그 장면에 메모 작성·저장. "내 메모 전체보기"에서 점프. 메모 있는 장면은 버튼에 점 표시.
- **장면별 대화 기억**: AI와 나눈 대화를 그 장면에 저장 → 재방문 시 참고("지난번에 물어보신 …").
- **저장 위치**: *현재* 이 브라우저(localStorage). **로그인 활성화 후** 사용자 계정에 묶여 **기기 간 클라우드 동기화**(대화는 요약 저장).
- ⚠️ 로그인 전에는 **브라우저 데이터를 지우면 기록도 사라집니다.**

## 8. 아키텍처 (플랫폼 연동)

정적 사이트는 비밀(API 키)·서버 로직을 숨길 수 없으므로, **Cloudflare Worker**(서버리스 백엔드)를 끼워 AI와 데이터를 처리합니다. 키·예산·검증은 전부 Worker 안에만 있고 브라우저로는 나가지 않습니다.

```
 [사용자 브라우저]
   프론트엔드 (GitHub Pages · 정적 HTML/JS)
   · 화면/애니메이션  · js/chat.js(AI질문)  · js/account.js(기록)
        │  질문+장면맥락 / 학습기록
        ▼
 [Cloudflare Worker]  ← 백엔드 (worker/eduviz-chat.js)
   · API 키·예산·토큰검증 보관     · 외부로 안 나감
        ├──→ Cloudflare Workers AI   (Llama 3.3 70B — 답변·요약 생성)
        ├──→ Cloudflare KV           (질문 예산 rpd/rpm · 사용자 기록 user:<id>)
        └──→ Google OAuth tokeninfo  (로그인 토큰 검증) [예정]
```

**데이터 흐름**
1. **AI 질문**: 브라우저 → Worker(POST `/`) → Workers AI → 답변 → 브라우저. 질문 수는 KV로 카운트.
2. **학습위치·메모·대화**: `account.js` → 로컬 저장(+ 로그인 시 Worker `?data=1` → 구글 토큰 검증 → KV `user:<id>`).
3. **로그인 전/후**: 전 = 이 브라우저에만 / 후 = 사용자별·기기 간 클라우드 동기화.

**주요 파일**
| 파일 | 역할 |
|---|---|
| `js/engine.js` | 공유 렌더·씬 엔진 |
| `js/content_*.js` · `content/*.json` | 트랙별 장면 동작·텍스트 |
| `js/physlab.js` | 물리 엔진 |
| `js/chat.js` | AI 질문 위젯 |
| `js/account.js` | 로그인·학습위치·메모·대화기억 |
| `js/chat-config.js` | 백엔드 주소·Client ID 설정 |
| `worker/eduviz-chat.js` | Cloudflare Worker(백엔드) 소스 |

## 9. 자주 묻는 질문 (FAQ)
- **Q. 로그인 필요한가요?** 아니요. 모든 학습 콘텐츠는 로그인 없이 열립니다. 로그인은 "기기 간 내 기록 동기화"를 원할 때의 선택지입니다.
- **Q. AI 답이 틀릴 수 있나요?** 네. 5.3 참조 — 중요한 내용은 화면의 실제 콘텐츠로 확인하세요.
- **Q. 비용이 드나요?** 사용자·운영 모두 무료(7장).
- **Q. 모바일도 되나요?** 됩니다(반응형). 다만 복잡한 시뮬은 큰 화면이 더 편합니다.
- **Q. 데이터(메모)가 사라졌어요.** 로그인 전엔 브라우저별 저장이라, 브라우저 데이터 삭제·다른 기기에선 보이지 않습니다.
